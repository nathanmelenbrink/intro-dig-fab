Images and Video in HTML

Taking Images and Video

Taking clear, representative images of your work is at least as important as writing clear and representative descriptions. Some low-effort tricks to get better images include:

  • Use a monotone background (white tables are great) free of clutter and other distractions.
  • Make sure you're zoomed out enough to capture the entire piece. You can decide later whether to crop, but you can't un-crop.
  • Hold your camera still, especially in low lighting or when taking a video. Feel free to use a tripod (there are a few in the lab) or clamp/helping hand to stabilize your camera.

Formatting Images and Video

Images and video that you take with your phone or DSLR camera are likely of far greater resolution (and size) than is needed for use on the web. This will slow down loading time for your pages considerably. Also, GitHub will not allow you to upload files greater than 100mb. Usually images of your work don't need to be more than a few hundred Kb each. In order to reduce the size of your files, you can use:

  • Online compressors like TinyPNG or TinyJPG (images)
  • Video compression. Mp4 seems to work best for hosting locally.
  • Software like Inkscape or Gimp.
  • Batch operations in Photoshop or similar (images only).

Hosting content on your page

Images and GIFs should be small enough to save in your local repository. This is also fine for short videos. The HTML should look something like this: 


<video controls style="width:50%; min-width:300px" >
	<source src="demo.mp4" type="video/mp4">
</video>

For longer videos, it often makes more sense to upload them to a streaming service like YouTube or Vimeo. Once you upload the video, you can right click and "copy embed code" to paste into your html file. It should look something like this: 


<iframe width="1019" height="573" src="https://www.youtube.com/embed/HBJMgkzkaas" frameborder="0" 
allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture" allowfullscreen> </iframe>

GIFs are effective ways to show short animations without taking up much space. You can convert a movie or series of images to a GIF using an online service like Giphy. GIFs can be referenced using the img tag, just like other images.

HTML Tags

For projects that involve programming, it's important to document your code in a clear and legible way. Writing comments in your code is always helpful (for others and for future you). Code should be documented using the proper HTML tags, never with screenshots.

For files that are too large to be reasonably pasted on your page, or for .zip folders containing multiple files, you are encouraged to make files downloadable. You do this by placing the .zip file (or any other kind of file) in your weekly directory and adding a link to it using the download attribute like this: 


<a href='./path-to-file' download > Download my file! <\a>

For short, in-line code snippets, you can use the HTML <code> tags, which looks something like this (depending on your CSS).

For multi-line excerpts, wrap the <code> tags in <pre> tags, like this: 


<pre><code>
void loop(){
  int i = 2;
  int j = 3;
}
</code></pre>

For even better legibility, you can use CSS libraries to style your code snippet to look like it does in the Arduino IDE. To do this, first paste these lines in the head section of your HTML page: 


<link rel="stylesheet" href="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.12.0/styles/arduino-light.min.css">
<script src="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.8.0/highlight.min.js"></script>
<script src="//cdnjs.cloudflare.com/ajax/libs/highlight.js/9.12.0/languages/arduino.min.js"></script>
<script>hljs.initHighlightingOnLoad();</script>

Then add the language-arduino class to your code tag (other languages have their own classes): 


<pre><code class="language-arduino">
void loop(){
  int i = 2;
  int j = 3;
}
</code></pre>

The result should look like this:


void loop(){
  int i = 2;
  int j = 3;
}
